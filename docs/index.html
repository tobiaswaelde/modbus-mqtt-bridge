Modbus MQTT Bridge

Connect Modbus TCP devices to MQTT with a lightweight Rust bridge.

Quick Start Configuration Topics Docker

Quick Start

  1. Clone the repository and edit config/config.yml.
  2. Start a local MQTT broker.
  3. Run cargo run -- --config config/config.yml.
  4. Subscribe to state topics and publish commands to /set topics.

Configuration Overview

The bridge reads YAML or JSON config. Default path is config/config.yml. If the file is missing, a template is generated automatically. 

mqtt:
  host: localhost
  port: 1883
  client_id: modbus-mqtt-bridge
  base_topic: modbus

sources:
  - id: example-device
    host: 127.0.0.1
    port: 502
    unit_id: 1
    poll_interval_ms: 2000
    request_timeout_ms: 3000
    points:
      - name: example_float
        topic: telemetry/example_float
        address: 0
        kind: holding
        data_type: f32
        access: read_write

MQTT Topic Contract

For base_topic=modbus, source_id=example-device, and topic=telemetry/example_float:

  • State topic: modbus/example-device/telemetry/example_float
  • Set topic: modbus/example-device/telemetry/example_float/set

State payloads are raw JSON values, for example:

21.5

Set payloads can be raw values or wrapped in an object:

{"value": 18.0}

Docker Deployment

Run the bridge with Docker Compose:

docker compose up --build

The container includes a healthcheck and uses a small runtime image for efficient deployment.
